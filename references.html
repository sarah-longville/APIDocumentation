Reference

An outline of all API functionality

Resources

There are 3 resources within the API to connect to; student, course and module. These can be all be accessed using an API console as mentioned on the home page.

GET Requests

A GET request will retrieve data from within the database. The request will then return the data in a JSON object.

PUT Requests

A PUT request can be used to edit current data within the database. The request will find the data from the request and replace it with the data entered into the body of the request.

POST Requests

A POST request is used to create new data within the database. This requests more than one piece of information. For example, if a new student was created the request would need to include the student name, course and student number

DELETE Requests

A DELETE request will delete data from within the database. Using this request you will need to be careful to only delete one data entry and not a whole table.

Student

Property Type Description
studentID int Unique identifier for each student generated every time a new student is added to the database
studentName varchar(50) Full name of the student
studentNumber int Unique student number assigned to each student when they enrol
course varchar(50) Name of course student is enrolled on

GET

GET
 
/student/{studentID}
 

GET
 
/student/{studentNumber}

Data can be retrieved from the student table either by the studentID or the studentNumber as these are both unique numbers. You need to add the appropriate studentID or studentNumber to the URI. The request could be sent using the student name but as multiple students may have the same name this could return multiple pieces of data. In order to return one student a unique field should be used in the request. The responses to these requests will return all of the data within the field as mentioned in the student table above.

Response

Student: {
            studentID: "1",
            firstName: "Sarah",
            lastName: "Longville",
            studentNumber: "10316502",
            course: "Web Applications Development",
            resource_uri: “http://fostvm.fost.plymouth.ac.uk/Modules/SOFT338/slongville/student/1”
}

PUT

PUT
 
/student/studentID/{courseName = name}
 

PUT
 
/student/studentNumber/{courseName = name}

Data can be edited from the student table either by the studentID or the studentNumber as these are both unique numbers. You need to add the appropriate studentID or studentNumber into the URI and write the name you want the details to be changed to. In carrying out these requests the information stored for a student will change. The request could be sent to just edit the students name but the students number could be edited too.

POST

POST
 
/student/{studentName = name}
This request will add a new student to the database. In order to do this studentNumber, course and name needs to be specified. The studentID is not needed as this is auto incremented. The response will then return the details of the new student.

Response

Student: {
	firstName: “Joanna”,
	lastName: “Lewis”,
	studentNumber: “10429839”,
	course: “Textiles”,
}

DELETE

DELETE
 
/student/{studentID}
 
DELETE
 
/student/studentNumber
These requests will delete a student from the database. In order to do this you will need to use the studentNumber or studentID. If the studentName is used this may delete more than one student so a unique identifier needs to be used. Make sure studentNumber or studentID is specified in the URI or this request will not work.

Course

Property Type Description
courseID int Unique identifier for each course generated every time a new course is added to the database
courseName varchar(50) Full name of the course
courseAcronym varchar(50) Acrocym assigned to each course name

GET

GET
 
/course/{courseID}
Data can be retrieved from the courses table by the courseID as this is the unique identifier. You need to add the appropriate courseID to the URI. The responses to these requests will return all of the data within the field as mentioned in the course table above.

Response

	Module: {
	moduleID: “1”,
	moduleName: “Advanced Web Technologies”,
	moduleCode: “ISAD334”,
	resource_uri: “http://fostvm.fost.plymouth.ac.uk/Modules/SOFT338/slongville/module/1”
}

PUT

PUT
 
/course/courseID/{courseName = name}
Data can be edited from the course table either by the courseID as this is the unique identifier. You need to add the appropriate courseID into the URI and write the name you want the details to be changed to. In carrying out this request the information stored for a course will change. The request could be sent to just edit the course name but the course acronym could be edited too.

POST

POST
 
/course/{courseName = name}
This request will add a new course to the database. In order to do this courseName, courseAcronym need to be specified. The courseID is not needed as this is auto incremented. The response will then return the details of the new course.

Response

	Module: {
	moduleName: “Internet Culture & Psychology”,
	moduleCode: “ISAD340”
}

DELETE

DELETE
 
/course/{courseID}
These requests will delete a course from the database. In order to do this you will need to use the courseID. Make sure the courseID is specified in the URI or this request will not work.
